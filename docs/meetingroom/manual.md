## 1. 引言

### 1.1 编写目的
本操作说明书旨在详细阐述《智能楼宇会议室系统 buildingos.meetingroom》的功能架构、操作流程及维护方法。文档面向系统的最终用户（包括企业员工、行政人员、会议室管理员），提供标准化的操作指引，帮助用户快速掌握从会议预约、审批、现场签到到会中设备智能控制的全流程操作，确保会议室资源的高效利用与会议体验的智能化。

### 1.2 定义
*   **系统/本系统**：指《智能楼宇会议室系统 buildingos.meetingroom》。
*   **中控平板**：指安装在会议室墙面或桌面的触控终端，用于集中控制会议室内的声、光、电及音视频设备。
*   **排期视图**：以时间轴或日历形式展示会议室占用情况的可视化界面。
*   **场景联动**：指一键触发多个设备的组合动作，如“投影模式”自动降下幕布、关闭主灯、开启投影仪。

## 2. 系统概述

### 2.1 系统用途
《智能楼宇会议室系统 buildingos.meetingroom》是专为现代智慧办公场景打造的综合性会议管理平台。系统集成了会议预约管理、信息发布、环境设备控制（IoT）及音视频会议控制功能。通过软硬件深度融合，解决传统会议室预定冲突、设备操作繁琐、资源闲置浪费等痛点，实现会议全生命周期的数字化管理。

### 2.2 软件功能概述
本系统主要包含以下核心功能模块：
1.  **资源可视化管理**：提供2.5D数字沙盘和日历排期视图，实时展示会议室占用状态。
2.  **全流程会议预约**：支持Web端/移动端预约，集成审批流、冲突检测及邮件/短信通知。
3.  **智能中控系统**：通过中控平板实现对灯光、窗帘、空调、大屏、音频等设备的集中控制。
4.  **视频会议集成**：深度对接腾讯会议、Zoom等主流会议软件，支持一键入会和会控操作。
5.  **数据统计分析**：自动统计会议室利用率、部门会议时长等数据，为行政决策提供依据。

### 2.3 软件运行环境
*   **硬件环境**：
    *   **服务端**：推荐配置 8核 CPU，16GB 内存，512GB SSD 存储。
    *   **中控端**：定制化安卓触控平板（10寸以上，PoE供电）。
    *   **外设支持**：兼容主流中控主机、SIP终端、继电器控制器及各类IoT传感器。
*   **软件环境**：
    *   **服务端系统**：Linux (Ubuntu 20.04+/CentOS 7+) 或 Windows Server 2019。
    *   **客户端**：支持 Chrome、Edge 等现代浏览器；中控APP支持 Android 10+。

## 3. 系统操作使用

### 3.1 登录与首页

#### 3.1.1 系统登录
系统采用B/S架构，用户通过浏览器访问指定网址即可进入登录界面。界面设计简洁商务，支持账号密码或SSO单点登录。
*   **操作步骤**：
    1.  输入系统URL地址，回车加载登录页。
    2.  在登录框中输入分配的用户名（Username）和密码（Password）。
    3.  点击“登录”按钮，系统进行身份验证，验证通过后跳转至首页。
*   **界面展示**：
    ![系统登录界面](image.png)
    *图 3-1 系统登录界面*

#### 3.1.2 系统首页概览
登录成功后进入系统首页（Dashboard）。首页作为工作台，聚合了会议管理的核心数据和快捷入口，帮助管理员一目了然地掌握今日会议动态。
*   **功能说明**：
    *   **数据看板**：顶部卡片显示“今日会议数”、“正在进行会议”、“空闲会议室”、“设备在线率”等关键指标。
    *   **快捷导航**：左侧菜单栏提供“会议室管理”、“会议预约”、“设备控制”、“统计报表”等功能模块的快速切换。
    *   **今日日程**：右侧列表展示当前登录用户的今日会议日程，点击可进入详情。
*   **界面展示**：
    ![系统首页](image-1.png)
    *图 3-2 系统首页概览*

### 3.2 会议室资源管理

#### 3.2.1 会议室数字沙盘
该模块通过2.5D平面图直观展示楼层会议室的分布及实时状态。
*   **功能说明**：
    *   **状态可视化**：会议室颜色代表状态（绿色-空闲、红色-占用、黄色-即将开始）。
    *   **快速预览**：鼠标悬停在会议室上方，显示当前会议主题、发起人及剩余时长。
    *   **环境监测**：实时显示会议室内的温度、湿度、PM2.5数值（需传感器支持）。
*   **界面展示**：
    ![会议室沙盘](image-2.png)
    *图 3-3 会议室数字沙盘*

#### 3.2.2 会议室排期视图
提供时间轴视角的会议室排期表，方便用户查找空闲时段。
*   **功能说明**：
    *   **多视图切换**：支持按日、周、月切换视图。
    *   **拖拽预约**：在空白时间段直接拖拽鼠标，即可快速发起该时间段的会议预约。
    *   **冲突提示**：若选择的时间段已有会议，系统会自动标红并禁止提交。
*   **界面展示**：
    ![预定会议](image-3.png)
    *图 3-4 会议室排期视图*

#### 3.2.3 新建会议预约
用户点击“新建会议”或在排期表中选择时段，进入会议预约详情页。
*   **功能说明**：
    *   **基本信息**：填写会议主题、时间、参会人数、会议类型（内部/外部）。
    *   **资源选择**：勾选所需会议室及配套服务（如茶水、投影、视频会议终端）。
    *   **参会人通知**：添加内部员工或外部访客邮箱，系统自动发送会议邀请函。
*   **界面展示**：
    ![新建会议](image-4.png)
    *图 3-5 新建会议预约*

#### 3.2.4 会议信息编辑
对于已创建但未开始的会议，发起人可进行修改或取消操作。
*   **功能说明**：
    *   **变更通知**：修改会议时间或地点后，系统自动触发变更通知发送给所有参会人。
    *   **会议纪要**：支持上传会议议程附件或预置会议纪要模板。
*   **界面展示**：
    ![编辑会议](image-5.png)
    *图 3-6 编辑会议信息*

#### 3.2.5 预定记录管理
管理员可查看系统中所有的历史预定记录，进行审计和异常处理。
*   **功能说明**：
    *   **状态追踪**：查看会议状态（待审批、已通过、进行中、已结束、已取消）。
    *   **审批管理**：对于需要审批的会议室，管理员在此进行通过或驳回操作。
    *   **签到记录**：查看参会人员的签到情况（支持扫码或人脸签到）。
*   **界面展示**：
    ![预定记录](image-6.png)
    *图 3-7 会议预定记录列表*

#### 3.2.6 会议时长统计
提供多维度的数据报表，帮助行政部门优化会议室资源配置。
*   **功能说明**：
    *   **时长趋势**：折线图展示日/周/月的会议总时长变化趋势。
    *   **部门排名**：统计各部门的会议时长和频次，评估部门协作活跃度。
    *   **利用率分析**：分析各会议室的忙闲时段，为后续扩容或缩减提供数据支撑。
*   **界面展示**：
    ![时长统计](image-7.png)

### 3.3 会议中控平板（智能控制）

    *   **快速签到**：支持扫描屏幕二维码或输入会议号进行现场签到。

    ![中控首页](image-8.png)

*   **功能说明**：
    *   **加入会议**：输入会议号一键加入腾讯会议/Zoom。

    *   **屏幕共享**：控制投屏内容的切换。
    *   **结束会议**：点击结束，系统自动释放会议室资源并重置设备状态。
